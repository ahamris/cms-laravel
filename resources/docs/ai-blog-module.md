# Inbound Marketing Orchestrator

## “Autopilot-first, Human-in-the-loop” Design

### Core Principle

> **Users describe intent, not tactics.**
> The system handles *what*, *when*, *where*, and *how often*.

---

## 1. From “Campaign Setup” → “Intent Brief”

Instead of asking marketing questions, ask **only 5 things**:

### Intent Brief (User Input)

* **Business goal**
  (More leads / Authority / SEO traffic / Product adoption)
* **Audience**
  (Decision-makers, developers, citizens, SMBs, etc.)
* **Topic / Problem**
  (“Woo compliance”, “AI in government”, etc.)
* **Tone**
  (Expert, neutral, persuasive)
* **Approval level**

  * Manual approval (default)
  * Auto-approve with spot checks

That’s it.

Everything else is inferred.

---

## 2. Marketing Intelligence Layer (New)

Introduce an **AI Marketing Brain** that decides:

* Best content types
* Posting cadence
* Channels
* Publish times
* Content mix (pillar → clusters → social)

### This layer does:

* SEO gap analysis (internal + external)
* Topic clustering
* Keyword intent classification
* Channel suitability scoring
* Timing optimization

The user never sees this unless they want to.

---

## 3. Autopilot Modes (Very Important)

### 3.1 Autopilot Levels

| Mode              | Description                            |
| ----------------- | -------------------------------------- |
| 🟢 Assisted       | AI proposes → user approves            |
| 🟡 Guided         | AI publishes unless blocked            |
| 🔴 Full Autopilot | AI plans, writes, schedules, publishes |

Default = **Assisted**

---

## 4. Smart Content Plan (Auto-Generated)

Instead of asking *what to generate*, the system creates a **Content Flight Plan**.

### Example Output (visible to user)

**Next 30 days plan**

* 1 Pillar article (SEO)
* 3 Supporting blog posts
* 12 Social posts (repurposed)
* 1 Evergreen update (existing content refresh)

User can:

* Accept all
* Remove items
* Change priority

No manual planning.

---

## 5. “When to Post” – Fully Automated

### How timing is decided

* Google Search Console data (if available)
* Historical CMS traffic
* Industry benchmarks
* Channel-specific best practices
* Timezone-aware

### Example logic

* Blog: Tue–Thu 09:00
* LinkedIn: Tue/Wed 08:30
* X: Mon–Fri 12:00 or 17:00
* Evergreen refresh: low-traffic hours

**User never picks a time.**

---

## 6. Single Approval, Not Per Post

Instead of approving every item:

### Approval happens at **Plan Level**

* Approve the content plan
* AI executes the plan
* User can still intervene per item

This removes friction.

---

## 7. Continuous Learning Loop

After publishing, the system learns:

* CTR
* Impressions
* Engagement
* Ranking changes

And adjusts:

* Headlines
* Length
* Posting time
* Internal linking
* Content depth

The system **improves itself** over time.

---

## 8. “Don’t Make Me Think” UI

### Main Dashboard shows only:

* 📈 Growth status
* 🗓 Next publications
* ⏳ Items awaiting approval
* ⚠️ Risks / suggestions

No SEO jargon unless expanded.

---

## 9. SEO: Move From “Fields” → “Guarantees”

Don’t show:

* Meta title fields
* Schema toggles
* Keyword density

Instead show:

* ✅ Google-friendly
* ⚠️ Needs improvement
* 🚀 High ranking potential

Under the hood:

* E-E-A-T
* Helpful Content
* Structured Data
* Internal links
* Canonicals
* Performance budget

---

## 10. Architecture Upgrade (Key Insight)

Add **two new services**:

### 1. Strategy Engine

* Decides *what* and *why*

### 2. Execution Engine

* Writes, schedules, publishes

The CMS becomes the **control plane**, not the brain.
